# MiniCar103 蓝牙功能实现与错误修复报告

## 概述
本报告记录了为MiniCar103项目添加蓝牙串口打印功能过程中遇到的问题、修复方案及详细修改内容。主要实现了通过BT24蓝牙模块将红外传感器数据和电机转速数据以JSON格式实时输出到电脑的功能，并支持远程控制命令。

**日期**：2026年4月6日  
**项目版本**：MiniCar103-v1.5.1  
**修复版本**：V1.5.2  
**作者**：Claude Code (Anthropic)

## 项目背景
MiniCar103是一个基于STM32F103C8T6的线跟随小车项目，采用时间片轮询调度系统。原项目包含8路红外传感器、PID循迹控制、OLED显示等功能。本次任务要求：
1. 通过蓝牙模块实时传输传感器和电机数据
2. 支持JSON格式数据输出
3. 支持远程命令控制（PID参数调整、小车控制等）

## 实现的功能特性

### 蓝牙数据传输
- **传输频率**：默认50ms（20Hz），可动态调整（10-1000ms）
- **数据格式**：JSON格式，便于程序解析
- **传输内容**：
  - 8路红外传感器状态
  - 左右电机速度（PWM占空比0-999）
  - PID误差值和积分值
  - 循迹使能状态和速度模式
  - 提前量补偿值

### 命令控制功能
- **数据传输控制**：启用/禁用传输，设置传输频率
- **状态查询**：获取当前状态和PID参数
- **PID参数调整**：动态设置Kp、Ki、Kd值
- **小车控制**：循迹使能/禁用，速度模式设置，手动控制电机

## 修复的编译错误

### 1. TaskMain.c 任务表注释乱码错误
- **错误信息**：`TaskMain.c(70): error: #146: too many initializer values`
- **问题根源**：中文注释在编译器中导致解析错误
- **修复方案**：将所有任务表中的中文注释改为英文注释
- **影响行**：第57-70行任务定义表

### 2. UsrTimer.c abs函数隐式声明警告
- **错误信息**：`UsrTimer.c(138): warning: #223-D: function "abs" declared implicitly`
- **问题根源**：abs()函数未正确声明，stdlib.h包含问题
- **修复方案**：将`abs(current_error)`改为`(current_error > 0 ? current_error : -current_error)`
- **影响行**：第138行

### 3. UsrTimer.c 变量赋值表达式错误
- **错误信息**：
  ```
  UsrTimer.c(241): error: #29: expected an expression
  ::leftSpeed = leftSpeed;
  UsrTimer.c(242): error: #29: expected an expression  
  ::rightSpeed = rightSpeed;
  ```
- **问题根源**：C语言不支持::作用域解析符，变量命名冲突
- **修复方案**：
  1. 将局部变量重命名为`leftSpeedLocal`和`rightSpeedLocal`
  2. 修改赋值语句为`leftSpeed = leftSpeedLocal;`
- **影响行**：第232-242行

### 4. oledfont.h 编码和语法错误（20个错误）
- **主要错误**：
  - `error: #8: missing closing quote` - 中文字符导致引号闭合错误
  - `error: #144: a value of type "char *" cannot be used to initialize an entity of type "unsigned char"` - 类型不匹配
  - `error: #1514: an empty initializer is invalid for an array with unspecified bound` - 空数组初始化语法错误
  
- **修复方案**：
  1. 将文件头注释、技术说明注释全部改为英文
  2. 将中文字符索引改为ASCII字符（"."、"1"、"2"、"3"、"4"、"5"）
  3. 注释掉未使用的空数组定义`Hzk1`、`Hzk2`、`Hzk3`、`Hzk4`
  4. 确保所有字符串使用正确的编码格式

## 新增功能实现

### 1. JSON数据格式
```json
{
  "type": "DATA",
  "tick": 123456,
  "sensors": [1,0,1,0,1,0,0,1],
  "baseSpeed": 600,
  "leftSpeed": 450,
  "rightSpeed": 550,
  "error": 10,
  "integral": 5,
  "advance": 2,
  "trackEnable": 1,
  "speedMode": 1
}
```

### 2. 支持的命令列表
1. **数据传输控制**：
   - `{"cmd":"ENABLE"}` - 启用数据传输
   - `{"cmd":"DISABLE"}` - 禁用数据传输
   - `{"cmd":"SETFREQ","value":50}` - 设置传输频率

2. **状态查询**：
   - `{"cmd":"STATUS"}` - 获取当前状态
   - `{"cmd":"HELP"}` - 显示帮助信息

3. **PID参数调整**：
   - `{"cmd":"SETPID","kp":15.0,"ki":0.0,"kd":5.0}`
   - `{"cmd":"GETPID"}`

4. **小车控制**：
   - `{"cmd":"TRACK","enable":1}` - 启用/禁用循迹
   - `{"cmd":"SPEED","mode":1}` - 设置速度模式
   - `{"cmd":"MANUAL","left":400,"right":400}` - 手动控制

## 已知限制与注意事项

### 1. 编码问题
- 项目中部分文件仍包含中文注释，未来修改时需注意编码问题
- 建议将项目所有源文件统一保存为UTF-8 without BOM格式

### 2. 实时性限制
- 蓝牙数据传输频率不建议低于10ms（100Hz）
- 高频传输可能影响其他任务执行

### 3. 内存限制
- STM32F103C8T6仅有20KB RAM
- 新增功能后需监控内存使用

### 4. 错误处理
- JSON解析错误时返回错误信息
- 无效命令返回`{"error":"Invalid command"}`
