# 最简智能小车
围绕教学用最简光电循迹小车的软硬件参考设计，设计目标如下：  
- 1、最简硬件结构：MCU最小系统/核心板、8路光电检测输入、2路直流电机驱动输出、OLED屏、2个按键、1个电源灯+1个功能灯、2节锂电池供电+电池电压监测、UART1转USB+UART3转蓝牙调测端口；  
- 2、基于时间片轮询的多任务程序框架，构建易于移植的智能小车软件体系：  
	- 2.1、基于时间片轮询程序框架满足对底层刨根究底的电类学生的教学（对软件不熟悉的电类学生，围绕基于RTOS的系统构建存在一定的教学难度）；  
	- 2.2、相对稳定可预测的程序框架，不因任务变更导致代码执行逻辑发生重大变化，适合于移植到很多其他小型应用，甚至开发严格实时系统；  
	- 2.3、进一步隔离系统移植层（基于CubeMX的代码移植）、应用任务层（基于时间片之上的任务）、硬件抽象层（硬件驱动、底层基于定时器触发的高频应用）  
- 3、易于以本代码为基础构建合理的嵌入式系统教学方法。

## 硬件设计汇总
本设计是围绕光电循迹小车的硬件，开展教学用最简智能小车的设计，具体特征如下：  
- 1、采用基于STM32F103C8T6的核心板作为主处理器  
	- a、基于Cortex-M3@72MHz的处理器内核  
	- b、内置64KB FLASH、20KB RAM  
- 2、外设汇总  

|序号|  功能  |  管脚  |  应用  |
|----|--------|--------|--------|
| 1  | GPIO  | PC13 | 核心板LED |
| 2  | GPIO  | PA11 | 底板LED |
| 3  | GPIO  | PC14 | KEY0 |
| 4  | GPIO  | PC15 | KEY1 |
| 5  | GPIO  | PB3  | 蜂鸣器 |
| 6  | UART1 | PA9/PA10 | USB转UART调试口 |
| 7  | UART3 | PB10/PB11 | 蓝牙传输UART |
| 8  | GPIO  | PB8/PB9  | OLED数据线 |
| 9  | GPIO  | PA0/1/2/3/4/5/6/7 | 光电信号输入 |
| 10 | ADC1IN8 | PB0 | 电池电压采样 |
| 11 | TIM3 CH1/2 | PB4/PB5 | 电机A路驱动 |
| 12 | TIM4 CH1/2 | PB6/PB7 | 电机B路驱动 |
| 13 | GPIO  | PB12/PB13 | 电机A路霍尔输入 |
| 14 | GPIO  | PB14/PB15 | 电机B路霍尔输入 |


## 软件框架设计汇总
- 1、采用时间片轮询的多任务框架  
	- a、采用SysTick作为时间片计时来源，以1ms为时基的一片时间片；  
	- b、所有修改在原有CubeMX生成代码内修改，main函数的while循环修改、SysTick中断函数修改；  
	- c、在main函数while循环之前增加BoardInit函数引领板级HAL初始化；  
- 2、板级HAL抽象层Board.c  
	- a、将所有板级HAL抽象层进行二次封装，与CubeMX代码隔离  
- 3、任务级应用层TaskMain.c  
	- a、所有任务调度放在任务表中静态初始化  
	- b、业务任务均由该文件引领  
- 4、具体业务描述  
	- a、按键检测  
		- i/ 采用多次检测方式，持续20ms以上确认为可靠的按键  
		- ii/ HAL层提供按键宏定义  
	- b、LED  
		- i/ HAL层提供相应操作的宏定义函数  
	- c、UART  
		- i/ UART1/UART3两个UART实现  
		- ii/ 采用FIFO发送/接收处理（非阻塞模式）  
		- iii/ 具体底层硬件发送/接收由中断服务函数完成  
	- d、printf函数封装  
		- i/ 由fputc重定义封装，目前在UART3输出  
	- e、OLED屏  
		- i/ 采用硬件I2C接口实现通信  
		- ii/ 帧存扫描传输方式  
		- iii/ 4ms一个page扫描  
	- f、电机驱动  
	- g、电机运行检测  
	- h、电池电压检测
