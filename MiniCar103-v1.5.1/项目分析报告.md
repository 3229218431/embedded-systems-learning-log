## 项目概述

### 项目背景

MiniCar103是一款面向高校教学的**最简智能光电循迹小车**参考设计，由浙江工业大学信息科学与工程学院人工智能专业开发。该项目旨在通过简单的硬件平台，帮助学生理解嵌入式系统开发的完整流程。

### 设计目标

1. **最简硬件结构**
   - MCU：STM32F103C8T6（Cortex-M3，72MHz）
   - 8路光电传感器输入
   - 2路直流电机PWM驱动输出
   - OLED显示、2个按键、蜂鸣器
   - 电池电压监测

2. **清晰的软件架构**
   - 基于时间片轮询的多任务框架
   - 硬件抽象层（BSP）与应用层隔离
   - 易于移植和扩展的学习平台

3. **教学友好性**
   - 避免RTOS的复杂性，降低入门门槛
   - 时间片调度逻辑透明可见
   - 代码注释详细，便于理解

### 时间片轮询调度原理

#### 调度器核心数据结构

```c
// TaskMain.h
typedef struct {
    unsigned char Run;        // 运行标志，0-未运行，1-运行就绪
    unsigned short Timer;     // 当前剩余定时器片值
    unsigned short ItvTime;   // 任务调度周期（毫秒）
    void (*TaskHook)(void);   // 任务函数指针
} TASK_COMPONENTS;
```

#### 调度流程详解

**步骤1：SysTick中断（1ms触发）**

```c
// stm32f1xx_it.c
void SysTick_Handler(void)
{
    HAL_IncTick();  // HAL库滴答计数递增
    
    // 每1ms递减所有任务的定时器片值
    for(i = 0; i < TASK_MAX; i++) {
        if(TaskComps[i].Timer) {
            TaskComps[i].Timer --;
            
            // 定时器片值归零时
            if(TaskComps[i].Timer == 0) {
                TaskComps[i].Timer = TaskComps[i].ItvTime;  // 重载
                TaskComps[i].Run = 1;       // 设置任务运行标志
            }
        }
    }
}
```

**步骤2：主循环扫描**

```c
// main.c
int main(void)
{
    // ... 系统初始化 ...
    
    while (1) {
        // 扫描所有任务
        for(i = 0; i < TASK_MAX; i++) {
            if(TaskComps[i].Run) {
                TaskComps[i].TaskHook();    // 调用任务函数
                TaskComps[i].Run = 0;       // 清除运行标志
            }
        }
    }
}
```

#### 任务表配置

```c
// TaskMain.c
TASK_COMPONENTS TaskComps[TASK_MAX] = {
    // 按键扫描任务 - 高优先级响应
    {0, 1, 4, ChkKey0Func},        // 按键0：4ms周期
    {0, 1, 4, ChkKey1Func},        // 按键1：4ms周期
    
    // 显示刷新任务 - 中等优先级
    {0, 5, 10, OLED_Update_InPages}, // OLED分页刷新：10ms
    
    // 通信任务 - 中等优先级
    {0, 3, 100, BuzzFunc},         // 蜂鸣器：100ms
    {0, 4, 100, Uart1Func},        // UART1处理：100ms
    {0, 4, 100, Uart3Func},        // UART3处理：100ms
    
    // 传感监控任务 - 中等优先级
    {0, 4, 100, ChkSenFunc},       // 传感器监控：100ms
    
    // 电池采集任务 - 低优先级
    {0, 4, 500, AdcReadTask},      // 电压采集：500ms
    
    // 指示灯任务 - 低优先级
    {0, 3, 1000, Led1Func},        // LED1闪烁：1000ms
    {0, 3, 2000, Led2Func},        // LED2闪烁：2000ms
    
    // 调试输出任务 - 低优先级
    {0, 3, 1000, PntFunc},         // 打印计数：1000ms
};
```

#### 调度器特点

| 特性 | 说明 |
|------|------|
| **优先级** | 任务在数组中越靠前，响应越快（受主循环扫描顺序影响） |
| **抖动** | 最大调度抖动为最长任务的执行时间 |
| **实时性** | 高频任务（4ms）可保证实时性 |
| **扩展性** | 增加任务只需修改TaskComps[]和TASK_MAX |

## 核心模块分析

### 1. 任务调度模块（TaskMain.c）

#### 按键扫描任务（ChkKey0Func）

```c
void ChkKey0Func(void)
{
    static uint8_t Cont = 0;        // 连续检测计数器
    static uint8_t key_pressed = 0; // 按键按下标志
    static uint8_t initialized = 0; // 初始化标志
    
    // 首次执行时显示初始状态
    if (!initialized) {
        initialized = 1;
        OLED_ShowString(96, 0, g_track_enable ? "ON " : "OFF", OLED_8X16);
    }
    
    if(GetKey0Val()) {  // 按键按下（低电平）
        Cont ++;         // 连续检测计数器增加
    } else {
        // 确认按键有效（消抖）：持续4次以上检测到按下
        if(Cont > 4 && !key_pressed) {
            key_pressed = 1;
            g_track_enable = !g_track_enable;  // 切换循迹使能
            // ... OLED更新 ...
        } else if (Cont <= 4) {
            key_pressed = 0;  // 短按或抖动
        }
        Cont = 0;  // 重置计数器
    }
}
```

**消抖原理：**
- 每4ms扫描一次按键
- 连续检测到按下>4次（即>16ms）才确认有效
- 有效按键后需松开（检测到高电平）才能再次触发

#### 速度模式切换任务（ChkKey1Func）

```c
void ChkKey1Func(void)
{
    static uint8_t g_speed_mode = 1;  // 默认中速
    
    if(Cont > 4 && !key_pressed) {
        key_pressed = 1;
        g_speed_mode = (g_speed_mode + 1) % 3;  // 循环切换：0→1→2→0
        
        switch(g_speed_mode) {
            case 0: baseSpeed = 400; break;  // 低速：约40% PWM
            case 1: baseSpeed = 600; break;  // 中速：约60% PWM（默认）
            case 2: baseSpeed = 800; break;  // 高速：约80% PWM
        }
    }
}
```

#### OLED分页刷新任务（OLED_Update_InPages）

```c
void OLED_Update_InPages(void)
{
    static uint8_t page = 0;  // 当前刷新的页号
    
    OLED_UpdateArea(0, page * 8, 128, 8);  // 刷新指定页面
    
    page++;
    if(page >= 8) {
        page = 0;  // 循环刷新8页
    }
}
```

**分页刷新优势：**
- 每次只刷新一页（8行），减少单次耗时
- 8次刷新完成完整画面（8 pages × 10ms = 80ms）
- 避免单次刷新阻塞其他任务

### 2. UART通信模块（uart.c）

#### FIFO结构体设计

```c
// FIFO.h - 通用FIFO
typedef struct {
    uint8_t *buffer;        // 缓冲区指针
    size_t size;            // 缓冲区大小
    size_t head;            // 读指针
    size_t tail;            // 写指针
    size_t count;           // 当前数据量
} fifo_t;

// uart.h - UART FIFO句柄
typedef struct {
    UART_HandleTypeDef *huart;
    fifo_t tx_fifo;         // 发送FIFO
    fifo_t rx_fifo;         // 接收FIFO
    uint8_t tx_buf[USART_TRS_LEN];  // 发送缓冲区
    uint8_t rx_buf[USART_REC_LEN];  // 接收缓冲区
    volatile bool tx_busy;  // 发送忙碌标志
} uart_fifo_handle_t;
```

#### 接收流程（中断方式）

```
UART中断触发
    ↓
读取RxByte
    ↓
写入rx_fifo
    ↓
重新启动中断（等待下一字节）
```

```c
// uart.c - 接收完成回调
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart)
{
    if (huart == g_uart1.huart) {
        uint8_t rx_byte = Rx1Byte;
        fifo_write(&g_uart1.rx_fifo, rx_byte);
        HAL_UART_Receive_IT(huart, &Rx1Byte, 1);  // 重启中断
    }
}
```

#### 发送流程（FIFO + 中断）

```
写入数据到tx_fifo
    ↓
检查tx_busy标志
    ↓
若空闲：启动HAL_UART_Transmit_IT
    ↓
发送完成回调：检查FIFO中是否还有数据
    ↓
有数据：继续发送下一批
无数据：清零tx_busy
```

```c
// uart.c - 发送
int uart_fifo_put(uart_fifo_handle_t *handle, uint8_t ch) {
    if (fifo_is_full(&handle->tx_fifo)) {
        return -1;  // FIFO已满
    }
    fifo_write(&handle->tx_fifo, ch);
    
    if (!handle->tx_busy) {
        handle->tx_busy = true;
        uint8_t data;
        fifo_read(&handle->tx_fifo, &data);
        HAL_UART_Transmit_IT(handle->huart, &data, 1);
    }
    return 0;
}
```

#### printf重定向

```c
// 支持标准库printf函数
int fputc(int ch, FILE *f)
{
    uart_fifo_put(&g_uart3, (uint8_t)ch);
    return ch;
}
```

#### 速度控制函数

```c
void Motor_SetSpeed(MotorControl_t *motor, int16_t speed_percent)
{
    // 限制范围：0-999
    if(speed_percent < 0) return;
    if(speed_percent > 999) speed_percent = 999;
    
    // 设置PWM占空比
    __HAL_TIM_SET_COMPARE(motor->pwm_timer, motor->pwm_channel1, speed_percent);
}
```

**Speed与PWM占空比：**
```
speed_percent = 0   →  PWM占空比 0%   → 电机停止
speed_percent = 400 →  PWM占空比 40%  → 低速运行
speed_percent = 600 →  PWM占空比 60%  → 中速运行
speed_percent = 800 →  PWM占空比 80%  → 高速运行
```

### 4. PID循迹控制模块（UsrTimer.c）

#### PID算法实现

```c
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM1) {
        // 1. 检查循迹使能
        if (!g_track_enable) {
            Motor_SetSpeed(&motor_left, 0);
            Motor_SetSpeed(&motor_right, 0);
            integral = 0;  // 重置积分项
            lastError = 0;
            return;
        }

        // 2. 读取8个传感器
        int sensor[8] = {
            GetSen0Val(), GetSen1Val(), GetSen2Val(), GetSen3Val(),
            GetSen4Val(), GetSen5Val(), GetSen6Val(), GetSen7Val()
        };

        // 3. 计算加权误差
        int weights[8] = {-2, -1, -1, 0, 0, 1, 1, 2};
        int error = 0;
        int activeCount = 0;

        for (int i = 0; i < 8; i++) {
            if (sensor[i]) {  // 检测到黑线
                error += weights[i];
                activeCount++;
            }
        }

        // 4. 没有检测到线的处理
        if (activeCount == 0) {
            Motor_SetSpeed(&motor_left, 0);
            Motor_SetSpeed(&motor_right, 0);
            integral = 0;
            lastError = 0;
            return;
        }

        // 5. PID计算
        integral += error;
        integral = clamp(integral, -1000, 1000);  // 积分限幅
        
        int derivative = error - lastError;
        lastError = error;
        
        int correction = (int)(Kp * error + Ki * integral + Kd * derivative);

        // 6. 速度计算
        int leftSpeed  = baseSpeed - correction;
        int rightSpeed = baseSpeed + correction;

        // 7. 速度限幅
        leftSpeed  = clamp(leftSpeed,  0, 999);
        rightSpeed = clamp(rightSpeed, 0, 999);

        // 8. 设置电机速度
        Motor_SetSpeed(&motor_left, leftSpeed);
        Motor_SetSpeed(&motor_right, rightSpeed);
    }
}
```

#### 误差计算原理

**传感器权重分配：**
```
位置：  Sen0  Sen1  Sen2  Sen3  Sen4  Sen5  Sen6  Sen7
权重：   -2    -1    -1     0     0    +1    +1    +2

中心点位于Sen3和Sen4之间（权重0）
```

**误差含义：**
- error < 0：小车偏左，需向右修正（右轮加速）
- error > 0：小车偏右，需向左修正（左轮加速）
- error ≈ 0：小车居中，直线行驶

**示例场景：**

| 场景 | 传感器状态 | error计算 | 修正方向 |
|------|------------|-----------|----------|
| 居中 | 00011000 | 0+0+0+0+0+0+0+0 = 0 | 直行 |
| 偏左 | 01110000 | -1-1+0+0+0+0+0+0 = -2 | 右转 |
| 偏右 | 00001110 | 0+0+0+0+1+1+1+0 = 3 | 左转 |
| 严重左偏 | 11110000 | -2-1-1+0+0+0+0+0 = -4 | 急右转 |
| 严重右偏 | 00001111 | 0+0+0+0+1+1+1+2 = 5 | 急左转 |

#### PID参数调优

```c
// UsrTimer.c - 默认参数
float Kp = 15.0f;   // 比例系数
float Ki = 0.0f;    // 积分系数（暂未使用）
float Kd = 5.0f;    // 微分系数

int baseSpeed = 600;  // 基础速度
```

**参数调整指南：**

| 参数 | 影响 | 调整策略 |
|------|------|----------|
| Kp | 响应速度 | 过大→振荡，过小→响应慢 |
| Ki | 消除稳态误差 | 可设为0，通过Kp调节 |
| Kd | 抑制振荡 | 过大→对噪声敏感，过小→稳定性差 |

**典型调参步骤：**
1. 先关闭Kd，调整Kp使小车能循迹
2. 逐渐增加Kd，消除左右摆动
3. 适当调整baseSpeed，找到最佳速度点

## 常见问题解答

### Q1：为什么按键扫描周期设为4ms？

**A：** 按键消抖需要一定时间，4ms周期可以：
1. 在16ms内完成4次检测（4×4ms=16ms）
2. 确保快速响应用户操作
3. 避免漏检短暂按键

### Q2：PID控制为什么放在TIM1中断中？

**A：** 放在中断中有以下优势：
1. 保证1ms的固定采样周期
2. 避免主循环任务阻塞导致控制延迟
3. 实时性更好，循迹更稳定

### Q3：如何增加新的任务？

**A：** 按以下步骤操作：

1. 在TaskMain.h中添加函数声明：
```c
void NewTaskFunc(void);
```

2. 在TaskMain.c中实现任务函数：
```c
void NewTaskFunc(void)
{
    // 任务代码
}
```

3. 在TaskComps[]数组中添加任务：
```c
TASK_COMPONENTS TaskComps[TASK_MAX] = {
    // ... 现有任务 ...
    {0, 1, 100, NewTaskFunc},  // 新任务：100ms周期
};
```

4. 如果任务数超过TASK_MAX，需要修改定义：
```c
#define TASK_MAX 12  // 原来是11
```

### Q4：小车循迹时出现振荡如何解决？

**A：** 可能原因及解决方案：

| 原因 | 现象 | 解决方案 |
|------|------|----------|
| Kp过大 | 小车左右剧烈摆动 | 减小Kp值 |
| Kd过小 | 系统响应迟钝 | 增加Kd值 |
| 传感器间隔过大 | 偏差检测不连续 | 减小传感器间距 |
| 基础速度过高 | 电机响应跟不上 | 降低baseSpeed |
| 地面反光 | 传感器误判 | 遮挡环境光 |

### Q5：OLED显示乱码如何解决？

**A：** 检查以下几点：

1. I2C引脚是否正确（PB8-SCL，PB9-SDA）
2. 上拉电阻是否连接（4.7kΩ）
3. OLED地址是否正确（0x3C或0x3D）
4. 供电电压是否正常（3.3V）

### Q6：为什么printf输出不到串口？

**A：** 检查以下配置：

1. 确认`fputc`函数已实现（uart.c中）
2. 确认UART3已正确初始化
3. 确认串口助手设置正确（115200 8N1）
4. 检查硬件连接（PB10-TX，PB11-RX）

### Q7：如何切换调试串口到UART1？

**A：** 修改uart.c中的fputc函数：

```c
int fputc(int ch, FILE *f)
{
    // 修改为g_uart1
    uart_fifo_put(&g_uart1, (uint8_t)ch);
    return ch;
}
```

### Q8：电机不转动如何排查？

**A：** 按以下步骤检查：

1. 检查电机电源是否连接
2. 检查PWM引脚电平是否变化（示波器）
3. 检查电机驱动芯片供电
4. 检查电机与驱动板连接
5. 在代码中直接设置PWM值测试：
```c
// 在BoardInit后添加临时测试代码
__HAL_TIM_SET_COMPARE(&htim3, TIM_CHANNEL_2, 500);
__HAL_TIM_SET_COMPARE(&htim4, TIM_CHANNEL_1, 500);
```
