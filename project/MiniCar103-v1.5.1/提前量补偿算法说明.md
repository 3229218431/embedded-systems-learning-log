# 提前量补偿算法说明

> 版本：V1.5.2  
> 修改时间：2026-04-01  
> 作者：AI Assistant  
> 目的：补偿红外传感器距离主动轮2cm造成的机械延迟

---

## 一、问题描述

### 1.1 机械布局问题

```
小车行进方向 →
  
  传感器阵列         主动轮
  [Sen0...Sen7]    (驱动轮)
       |              |
       |← 2cm延迟 →|
       |
  距离主动轮前方2cm处
```

### 1.2 问题现象

| 位置 | 传感器检测 | 主动轮位置 | 问题 |
|------|-----------|-----------|------|
| t=0ms | 检测到弯道 | 尚未到达 | 正常 |
| t=40ms | 检测到弯道 | 即将通过 | **延迟40ms** |
| t=80ms | 检测到弯道 | 已经过弯 | **响应滞后** |

### 1.3 延迟计算

```
延迟距离：2cm = 0.02m
小车速度（中速）：约0.5m/s
延迟时间 = 0.02m / 0.5m/s = 40ms

PID控制周期：1ms (TIM1中断)
40ms内可执行40次PID计算
```

## 二、算法原理

### 2.1 核心思想

**预测控制**：根据当前误差和速度，预测未来位置的误差，提前执行转向操作。

```
当前误差 → 预测未来误差 → 计算提前量 → 补偿PID输出
```

### 2.2 数学模型

#### 2.2.1 延迟时间估计

```c
delay_time = delay_distance / (speed × speed_factor)
```

- `delay_distance`: 机械延迟距离（单位：0.1mm）
- `speed`: 当前速度百分比（0-999）
- `speed_factor`: 速度换算系数（约30对应3.0mm/ms）

#### 2.2.2 预测误差计算

```c
predicted_error = current_error × (1 + delay_factor)
advance_compensation = predicted_error - current_error
```

#### 2.2.3 动态调整

考虑速度变化率对预测的影响：
- **加速时**：增加提前量
- **减速时**：减少提前量

### 3.2 核心函数 (UsrTimer.c)

#### 3.2.1 初始化函数

```c
void AdvanceCompensation_Init(void)
{
    delay_distance = 200;      // 2.0cm = 20.0mm = 200 (0.1mm单位)
    
    /* 根据速度模式设置估计速度 */
    switch(g_speed_mode) {
        case 0: car_speed_estimate = 20; break;  // 低速 2.0mm/ms
        case 1: car_speed_estimate = 30; break;  // 中速 3.0mm/ms（默认）
        case 2: car_speed_estimate = 40; break;  // 高速 4.0mm/ms
    }
}
```

#### 3.2.2 提前量计算函数

```c
int16_t Calculate_Advance_Compensation(int16_t current_error, int16_t speed_percent)
{
    /* 计算延迟时间对应的位移（单位：0.1mm） */
    uint16_t delay_time_equivalent = delay_distance;
    
    /* 计算延迟导致的误差放大系数 */
    int32_t delay_comp_factor = (int32_t)delay_time_equivalent * 100 / (speed_factor * 10);
    
    /* 简化的预测公式：提前量 = 误差 × 延迟系数 */
    int32_t predicted = current_error + (current_error * advance_factor / 100);
    compensation = (int16_t)(predicted - current_error);
    
    /* 速度变化补偿 */
    if (speed_change_rate > 50) {
        compensation += compensation * speed_change_rate / 200;  // 加速增加提前量
    } else if (speed_change_rate < -50) {
        compensation += compensation * speed_change_rate / 200;  // 减速减少提前量
    }
    
    /* 限制补偿范围 */
    if (compensation > 300) compensation = 300;
    if (compensation < -300) compensation = -300;
    
    return compensation;
}
```

#### 3.2.3 带补偿的PID控制

```c
void PID_Control_With_Advance(int sensor[8], const int8_t weights[8])
{
    /* 1. 计算加权误差 */
    error = 0;
    for (int i = 0; i < 8; i++) {
        if (sensor[i]) error += weights[i];
    }
    
    /* 2. PID计算 */
    integral += error;
    integral = clamp(integral, -1000, 1000);
    derivative = error - lastError;
    lastError = error;
    correction = (int)(Kp * error + Ki * integral + Kd * derivative);
    
    /* 3. 计算并应用提前量补偿 */
    comp = Calculate_Advance_Compensation(error, baseSpeed);
    correction += comp;
    
    /* 4. 计算左右电机速度 */
    leftSpeed  = baseSpeed - correction;
    rightSpeed = baseSpeed + correction;
    
    /* 5. 限幅并设置 */
    // ...
}
```

### 4.3 参数调节步骤

#### 步骤1：测量实际延迟时间

```
1. 在直道上标记起点和终点
2. 测量传感器到主动轮的实际距离（例如2.0cm）
3. 根据速度估算延迟时间：
   - 低速（40%）：约50ms
   - 中速（60%）：约33ms
   - 高速（80%）：约25ms
```

#### 步骤2：观察小车行为

| 现象 | 原因 | 调整方法 |
|------|------|----------|
| 小车总是偏向一侧 | 提前量过大 | 减小 `advance_factor` |
| 小车响应迟钝 | 提前量不足 | 增加 `advance_factor` |
| 小车振荡 | 速度估计错误 | 调整 `car_speed_estimate` |
| 过弯时抖动 | 补偿 Chang铑ang_rate | 减小速度变化率补偿 |

#### 步骤3：精细调节

```c
/* 在UsrTimer.c中调节 */
int32_t delay_comp_factor = (int32_t)delay_time_equivalent * 100 / (speed_factor * 10);

/* 如果效果不明显，可以放大系数 */
delay_comp_factor = (int32_t)delay_time_equivalent * 150 / (speed_factor * 10);  // ×1.5

/* 如果过度补偿，可以缩小系数 */
delay_comp_factor = (int32_t)delay_time_equivalent * 75 / (speed_factor * 10);   // ×0.75
```

## 八、常见问题

### Q1：为什么小车还是有延迟？

**A:** 可能原因：
1. `delay_distance` 设置不准确，重新测量物理距离
2. `car_speed_estimate` 与实际速度不符，调整该参数
3. 提前量达到上限（±300），需要减小 `Kp` 或降低速度

### Q2：如何测试延迟补偿效果？

**A:** 测试方法：
1. 在直道上画一条明显的弯道标记
2. 观察小车到达弯道时的转向时机
3. 理想情况：在传感器刚检测到弯道时，小车已开始转向

### Q3：不同路面如何调整？

**A:** 建议参数组合：

| 路面类型 | delay_distance | speed_estimate | 说明 |
|----------|---------------|----------------|------|
| 光滑地面 | 200 | 30 | 标准设置 |
| 粗糙地面 | 200 | 25 | 速度较低 |
| 粘性地面 | 200 | 35 | 速度较高 |
