# Embedded Systems Learning Log

This repository documents my hands-on learning journey in **embedded systems**, **control engineering**, and **Edge AI**.

The goal of this repo is not to present polished products, but to record **real engineering practice**: experiments, simulations, design decisions, bugs, and lessons learned.

---

## 👤 About Me

- Undergraduate student majoring in Electronic Information Engineering
- Focus areas: **Embedded Systems · Control Systems · Edge AI**
- Interests: engineering-oriented learning, competitions, and open-source collaboration

I use this repository as a long-term learning log and technical portfolio.

---

## 📌 Repository Structure

```
embedded-systems-learning-log/
├── c-language/            # C language fundamentals for embedded systems
├── mcu-basics/            # MCU peripherals, drivers, and bare-metal practice
├── control-systems/       # Control logic (PID, state machines, etc.)
├── simulation-matlab/     # MATLAB / Simulink simulations and modeling
├── edge-ai/               # TinyML / Edge AI experiments
├── competitions/          # Preparation & projects for competitions (E-Design, Huawei, etc.)
├── README.md
├── LICENSE
└── .gitignore
```

Each folder may contain:
- Notes (`.md`)
- Source code (`.c`, `.h`, `.py`)
- Simulation files (`.slx`, `.m`)
- Experiment logs

## 🧭 Learning Philosophy

- **System first, details later**: prioritize making systems run end-to-end
- **Engineering over theory-only learning**
- **Simulation → implementation → iteration**
- Treat bugs and failures as part of the documentation
